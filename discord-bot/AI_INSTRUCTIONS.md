# 🤖 Family Legends - Discord Bot (AI Context & Rules)

هذا الملف مخصص لأي ذكاء اصطناعي (AI) يعمل على هذا المشروع لفهم البيئة، القواعد، وأهداف النظام لتجنب أي أخطاء مستقبلية وضمان تكامل المشروع بأعلى جودة.

## 🛠️ معلومات البيئة والهيكلة (Environment & Stack)
*   **نوع المشروع:** Discord Bot متطور (Node.js) يوفر خدمات (إدارة، اقتصاد، موسيقى، حماية Anti-Nuke، تذاكر).
*   **منصة الاستضافة (Deployment):** Render (Web Service).
*   **قاعدة البيانات الرئيسية:** MongoDB (مستضافة على MongoDB.com - عبر مكتبة Mongoose).
*   **الربط مع لوحة التحكم:** البوت يحتوي على خادم Express.js مصغر لاستقبال طلبات (HTTP POST/GET) من لوحة التحكم لتنفيذ أوامر مثل (إرسال رسائل، تعديل إعدادات..).
*   **التواصل مع الذكاء الاصطناعي:** يجب دائماً المخاطبة باللغة العربية، الشرح المبسط للمستخدم، والعمل كمهندس خبير (Senior Engineer).

## 🔒 القواعد الأمنية (Security Rules - CRITICAL)
1.  **المتغيرات البيئية (Env Variables):** يُمنع منعاً باتاً كتابة أي توكن (Discord Token) أو كلمات مرور (MongoDB URI, OpenAI, API_SECRET) بشكل صريح داخل الأكواد. جميع المتغيرات يجب أن تُستدعى عبر `process.env`.
2.  **رفع الملفات (Commits):** يجب التأكد دائماً أن ملفات `.env` موجودة في `.gitignore`. ولا يجب رفعها أبداً إلى GitHub.
3.  **المصادقة (Authentication):** أي واجهة برمجية (API Route) داخل سيرفر Express الخاص بالبوت يجب أن تكون محمية عبر `BOT_CONTROL_SECRET` القادم من Headers، ويُمنع الاعتماد على الثقة العمياء.

## 🌟 المعايير البرمجية والوظيفية
*   **الاستقرار (Stability):** يجب حماية البوت من التوقف المفاجئ باستخدام (Error Boundaries) مثل `unhandledRejection` في `index.js`.
*   **الأداء:** يجب تجنب الاستعلامات المتكررة لقاعدة البيانات (N+1 Query) واستخدام طرق التخزين المؤقت (Caching) حيثما أمكن.
*   **الصلاحيات (Permissions):** أي أمر أو إجراء حساس يجب فحصه (هل المستخدم لديه صلاحية `Administrator`؟ هل رتبة البوت كافية لتنفيذ الأمر؟).

## 🚀 GitHub & AI Permissions
لعشران تفعيل قدرة الذكاء الاصطناعي على حفظ التعديلات مباشرة:
1.  **المصادقة:** تم ضبط الـ Git Remote لاستخدام الـ Personal Access Token تلقائياً.
2.  **الأوامر:** يمكن استخدام `git commit` و `git push origin main` لحفظ التغييرات.
3.  **التوكينات:** يجب التأكد دائماً من وجود `GITHUB_TOKEN` في ملف `.env` وعدم رفعه.
4.  **Queen Coder:** تم تهيئة المشروع ليعمل بسلاسة مع أدوات مثل Queen Coder عبر الـ Remotes الموثقة.
